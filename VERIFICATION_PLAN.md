# AT32F456 FOC 硬件验证计划

## 前置条件

### 硬件准备
- AT32F456CEU7 驱动板（已焊接完成）
- BLDC电机（NPP=8 极对数）
- DPT双磁编码器（UART 2.5Mbps）
- 24V/48V直流电源（电流限制设为2A起步）
- USB-TTL串口线（连接USART1，PA9/PA10）
- 示波器（建议2通道以上）
- CAN分析仪（选做，可后续验证）

### 软件准备
- 串口终端（PuTTY/SecureCRT/VSCode Serial Monitor）
- 波特率：921600（USART1调试口）
- 当前固件：已编译通过（Code=59KB, 0 Error）

### 安全提醒
- ⚠️ 当前固件默认状态：`foc_run=0, openloop=0`，**电机不会动**
- ⚠️ 每次测试前确认电流限制设在安全范围
- ⚠️ 开环测试时电机可能突然转动，固定好电机
- ⚠️ EN_GATE (PC15) 控制栅极驱动使能，硬件急停最后防线

## 第一阶段：基础上电验证（无需电机转动）

### 1.1 纯MCU上电（不接电源母线）

**目的**：确认MCU启动、串口通信、FOC初始化流程正常

**操作**：烧录固件 → 接串口 → 上电

**预期串口输出**：
```
 FOC start
DWT: 1000-NOP cycles=1261
DWT: 100us busy-wait, actual=100 us
ADC: calibrating offsets (1024 samples)...
ADC: offset_a=XXX offset_b=XXX
Initializing FOC controller...
FOC init done: NPP=8 foc_run=0 mode=2
---- 1s status ----
INT: PCCE=10003 OCCE=1000 DMA=1000 TCF=0
ADC: i_a=0 i_b=0 (raw_a=XXX raw_b=XXX offs_a=XXX offs_b=XXX)
     temp_m=XXX temp_mos=XXX so_c=XXX so_3=XXX vdc=0
DPT: inner=XXX outer=XXX
FOC: run=0 mode=2 openloop=0
     I_d=0 I_q=0 I_q_ref=0 theta_e=0 pos_out=0 dtheta=0
```

**检查项**：
- [ ] DWT计时正确（1000-NOP≈1261cycles @192MHz）
- [ ] ADC offset校准完成（offset_a应在1900~2100范围，12bit中点）
- [ ] PCCE计数≈10000/s（10kHz注入采样）
- [ ] OCCE计数≈1000/s（1kHz普通组）
- [ ] DMA计数≈1000/s（DMA传输普通组数据）
- [ ] FOC初始化成功（NPP=8）
- [ ] i_a/i_b≈0（无电流时校准后接近零）
- [ ] DPT编码器有数据（inner/outer角度值）

**串口命令测试**：
```
version↵       → 应回显版本和编译时间
getparams↵     → 应打印NPP=8, PID参数等
logid110↵      → 设置日志模式为ISR耗时
logid0↵        → 关闭日志
```

**如果失败**：
- 无任何输出 → 检查串口连接（PA9=TX, PA10=RX），波特率921600
- 卡在"ADC: calibrating" → ADC注入组未触发，检查TMR1 TRGO配置
- PCCE=0 → ADC1 PCCE中断未使能或TMR1未启动
- DPT读取失败 → 检查USART3连接（PB10=TX, PA6=RX, PB14=DE）

---

### 1.2 接通母线电源（24V/48V）

**目的**：确认VDC采样、温度采样、电流零点

**操作**：接通直流电源（设电流限制1A）→ 观察串口

**预期变化**：
```
     temp_m=XXX temp_mos=XXX so_c=XXX so_3=XXX vdc=YYYY
```
vdc应为非零值（具体数值取决于分压比）。

**检查项**：
- [ ] vdc原始值非零（24V电源下应有明确读数）
- [ ] 电流i_a/i_b仍然≈0（电机未转，无电流流过）
- [ ] 温度raw值合理（室温下不应为0或4095满量程）
- [ ] 电源电流表读数<50mA（MCU+驱动IC静态功耗）

**VDC换算公式**（根据硬件分压比推算）：
```
实际电压 = vdc_raw × 3.3V / 4096 × 分压比
```
记录raw值和实际电压，后续闭环需要正确的UDC标定。

**如果失败**：
- vdc=0 → 检查VDC采样通道配置（普通组DMA是否正常）
- 电源电流异常大(>200mA) → 可能有短路，立即断电检查
- 温度值异常 → NTC电路检查（开路=4095，短路=0）

---

### 1.3 EN_GATE 使能测试

**目的**：确认DRV8353/栅极驱动IC正常唤醒

**操作**：
```
enable1↵       → 使能PWM输出（此时CCR=0，占空比=0，无电流）
```

**预期**：
- EN_GATE (PC15) 应为高电平
- DRV8353的nFAULT引脚应为高（正常）
- 电源电流略增（~10-20mA，驱动IC启动电流）

**用示波器验证**：
- 探头接 PWM_A (PA8) → 应看到10kHz中心对齐PWM，占空比≈50%
- 三相应该相同占空比（因为CCR=period/2, SVPWM零矢量）

**如果失败**：
- nFAULT为低 → DRV有故障（过流/欠压/过温），检查PVDD电源
- 无PWM波形 → TMR1->brk MOE位未置位，或BKIN锁死

## 第二阶段：编码器验证

### 2.1 DPT编码器连续读取

**目的**：确认编码器角度跟随物理转动

**操作**：手动转动电机轴 → 观察串口DPT数据

**预期**：
```
DPT: inner=XXX.XX outer=YYY.YY
```
- 转动电机轴，inner角度应连续变化（0→360循环）
- 正转/反转方向应对应角度增/减
- outer角度（如果输出端有减速器）变化幅度按减速比缩小

**检查项**：
- [ ] 角度范围0~360°完整覆盖
- [ ] 转动平滑，无跳变（CRC错误会导致跳变）
- [ ] 正反转方向正确

**串口命令辅助**：
```
logid11↵       → 持续打印编码器raw值
logfreq100↵    → 100ms一次
```

### 2.2 编码器分辨率确认

**操作**：缓慢转动一圈（360°），记录inner_raw变化范围

**预期**：
- inner_raw: 0 ~ 16777215 (24bit, 2^24-1)
- 分辨率: 360° / 16777216 ≈ 0.0000215°/LSB

## 第三阶段：开环测试（电机转动）

### 3.1 开环低压启动

**⚠️ 安全注意：电机会转动！固定好电机和线缆！**

**操作**：通过串口命令启动开环
```
openloop1↵     → 启用开环自动旋转模式
vq128↵         → 先用小电压 (Q10=128, 约12.5%最大电压)
```

**预期**：
- 电机开始缓慢旋转
- 串口输出（启用logid120）：
```
OpenLoop: theta=XXXXX I_a=XX I_b=XX I_c=XX V_d=0 V_q=128
```
- theta值周期性变化（0→65535循环）
- 电流I_a/I_b/I_c有非零值（开环产生的实际电流）
- 电源电流表显示<500mA

**检查项**：
- [ ] 电机转动平滑（不抖动、不堵转）
- [ ] 转速与v_q成正比（增大vq转更快）
- [ ] theta周期性循环（电角度扫描）
- [ ] 电流值合理（不应超过2A @128 Q10电压）

**如果失败**：
- 电机抖动不转 → 相序错误（交换任意两相线试试）
- 电机有嗡鸣但不转 → v_q太小，增大到256/512
- 过流保护触发 → 降低v_q，检查相线短路

### 3.2 开环方向和相序确认

**操作**：
```
vq128↵         → 正转
vq-128↵        → 反转（如果支持负值，或用vq65408表示-128的uint16）
```

**检查**：
- [ ] 正/负vq对应电机正/反转
- [ ] 编码器角度增长方向与电机旋转方向一致

**如果编码器方向和电机方向相反**：
- 需要在encoder_calc.c里取反角度
- 或者修改NPP为负值（参考工程的做法）

### 3.3 开环电流采样验证

**操作**：
```
logid90↵       → 打印三相电流raw
logfreq50↵     → 50ms一次
openloop1↵
vq256↵
```

**预期**：
```
Iraw: a=XXXX b=YYYY c=ZZZZ
```
- 三相电流应呈正弦变化（相位差120°）
- raw值围绕offset中心上下波动
- 波动幅度与v_q成正比

**检查项**：
- [ ] 三相电流幅值接近（不平衡<10%）
- [ ] 电流波形连续无跳变
- [ ] 停止后电流回归0

**目的**：确定编码器电角度零点与电机d轴的偏差

**前置条件**：
- 开环测试通过（相序正确、编码器方向正确）

controller_eyou.theta_elec = 0;  // 固定角度=0

- 电机会锁定在一个固定角度（d轴对齐）
- 读取此时编码器角度 → 就是elec_offset

## 第五阶段：闭环电流环验证

### 5.1 电流环基本跟踪

**前置条件**：
- 电角度已校准（elec_offset正确）
- PID参数已设置（参考默认值或参考工程值）

**操作**：
```
getparams↵                     → 确认当前PID
CurrentPID Kp100Ki50Kd0↵       → 设置电流环PID（保守值）
Run cmd1 M2 tar0.1↵            → 启动力矩模式，目标0.1Nm
logid40↵                        → 查看电流PI状态
logfreq100↵
```

**预期**：
```
CurPI: I_q=XXX I_d=0 V_q=YYY V_d=ZZZ I_qref=1024 I_dref=0
```
- I_q 应跟踪 I_q_ref（稳态误差<5%）
- I_d 应趋近 0（d轴电流解耦）
- 电机产生恒定扭矩（可手感受阻力）

**检查项**：
- [ ] I_q 稳态跟踪准确
- [ ] I_d ≈ 0
- [ ] V_q/V_d 在合理范围（不饱和）
- [ ] 无振荡/啸叫

**停止**：
```
Run cmd0 M2 tar0↵     → 或
enable0↵               → 紧急停机
```

### 5.2 电流环PID整定

**方法**：增大I_q_ref观察阶跃响应

```
Run cmd1 M2 tar0.5↵           → 0.5Nm
logid40↵
logfreq10↵                     → 10ms打印一次，观察阶跃
```

**调参步骤**：
1. 先只用P：`CurrentPID Kp200Ki0Kd0` → 看是否跟得上
2. 加I消除稳态误差：`CurrentPID Kp200Ki100Kd0`
3. 如果振荡 → 降Kp
4. 如果响应慢 → 增Kp或增Ki

**典型参考值**（NPP=8 小电机）：
- Kp: 100~500
- Ki: 50~200
- Kd: 0（电流环一般不用D）

### 5.3 电流环带宽测试

**操作**：阶跃I_q_ref从0到满载，测量上升时间

**指标**：
- 上升时间(10%~90%): 目标<1ms
- 超调: <10%
- 稳态误差: <2%

## 第六阶段：速度环验证

### 6.1 速度环基本跟踪

**前置条件**：电流环已整定好

**操作**：
```
SpeedPID Kp200Ki100Kd0↵        → 设置速度环PID
Run cmd1 M3 tar10↵             → 速度模式，目标10rpm
logid50↵                        → 查看速度环状态
```

**预期**：
```
Spd(rpm): ref=10 filt=10 act=9 err=1
```
- 电机以~10rpm稳定旋转
- 速度误差在±1rpm以内

**检查项**：
- [ ] 速度稳态跟踪
- [ ] 加减速平滑
- [ ] 不同速度都能跟踪（10/50/100/500rpm）

### 6.2 速度环负载测试

**操作**：手施加负载（捏住电机轴）

**预期**：
- 速度暂时下降 → 电流增大（I_q增加）→ 速度恢复
- 松开后速度回到给定值
- 过载时应触发过流保护

## 第九阶段：性能测量

### 9.1 ISR耗时测量

**操作**：
```
logid110↵      → ISR分段耗时
logfreq1000↵   → 1秒一次
```

**预期输出**：
```
ISR_us: read=X/X enc=X/X pos=X/X vel=X/X cur=X/X
```
格式: `last/max`，单位 us

**指标**（@192MHz, 10kHz采样, 周期=100us）：
- 总ISR耗时: 目标<80us（留20us裕量）
- 电流环: 目标<30us
- 编码器: 目标<20us（阻塞读取可能较慢）
- 位置环: 目标<10us（4拍一次）
- 速度环: 目标<10us（2拍一次）

**如果超过100us**：
- 编码器阻塞读取太慢 → 需要异步DMA方案
- 电流环计算太慢 → 开-O3优化 / 查表代替运算

### 9.2 中断优先级冲突检测

**观察**：
- PCCE计数是否严格=10000/s（少了说明中断被抢占太久）
- 串口是否有乱码（DMA优先级冲突）

## 故障排查速查表

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 无串口输出 | 串口线/波特率/引脚错误 | 示波器看PA9是否有波形 |
| 卡在ADC校准 | TMR1未启动/PCCE未使能 | 检查PCCE计数器 |
| vdc=0 | DMA未传输/ADC普通组未触发 | 检查DMA计数 |
| DPT读取失败 | USART3配置/DE引脚/波特率 | 示波器看PB10发送波形 |
| 开环电机不转 | 相序错/EN_GATE未使能/BKIN锁死 | 示波器看6路PWM |
| 开环电机抖动 | 相序错/编码器方向反 | 交换两相线重试 |
| 闭环振荡 | PID参数太大/电角度offset错 | 降Kp到10重试 |
| 过流保护 | 电流采样offset错/硬件短路 | 检查offset校准值 |
| Flash写入失败 | 地址错/sector大小错/中断打断 | 用logid162 dump检查 |
| CAN无数据 | 总线未连/终端电阻缺/波特率错 | CAN分析仪抓包 |
